```
{00_common.md 공통 구현 블록 붙여넣기}

[작업 지시]
PR-02: Supabase 연동 + 핵심 DB 스키마를 구현해줘.
현재 프로젝트 상태: PR-01 완료 (Expo 프로젝트 초기화)

[구현 단계]
1. npm install @supabase/supabase-js + npx expo install expo-secure-store
2. .env.local 생성 (EXPO_PUBLIC_SUPABASE_URL, EXPO_PUBLIC_SUPABASE_ANON_KEY), .gitignore 추가
3. src/services/supabase.ts: SecureStore adapter 토큰 영속화, 싱글톤 클라이언트
4. supabase/migrations/001_create_profiles.sql:
   CREATE TABLE profiles (
     id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
     username TEXT UNIQUE NOT NULL,
     nickname TEXT NOT NULL,
     phone TEXT UNIQUE NOT NULL,
     role TEXT NOT NULL DEFAULT 'user' CHECK (role IN ('super_admin','admin','user')),
     status TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active','suspended','deleted_pending')),
     created_at TIMESTAMPTZ DEFAULT now(),
     updated_at TIMESTAMPTZ DEFAULT now()
   );
   ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
   CREATE POLICY "profiles_select_own" ON profiles FOR SELECT USING (auth.uid() = id);
   CREATE POLICY "profiles_update_own" ON profiles FOR UPDATE USING (auth.uid() = id) WITH CHECK (auth.uid() = id);
5. src/types/database.ts: 타입 정의
6. src/services/profileService.ts: getProfile(userId), updateProfile(userId, data)
7. TDD: supabase 클라이언트 생성 테스트, profileService 모킹 테스트

[경계 조건]
- service_role 키 클라이언트에 절대 포함 금지
- RLS 비활성 테이블 금지
- INSERT 정책 없음 (트리거로 처리 예정)

[완료 조건]
- Supabase 클라이언트 초기화 성공
- profiles 테이블 + RLS 활성화
- profileService 모킹 테스트 통과
- TypeScript 타입 파일 존재
- .env.local이 .gitignore에 포함
```
